Computer Vision demos often run on \$2,000 NVIDIA GPUs. But in the real world—especially in IoT—you don't have that luxury. You have a Raspberry Pi 4 with 4GB of RAM and no dedicated GPU.

This article documents my journey to optimize a "Home Surveillance System" to run locally without cloud dependency. The goal? Detect "Person" and "Dog" at 30 FPS.

1. The CPU Bottleneck

Running a standard `YOLOv5s` model on a Raspberry Pi CPU results in ~1-2 FPS. This is unusable for security monitoring where an intruder can run past the camera in the 2 seconds between frames.

The Solution: MobileNet SSD

Single Shot Detectors (SSD) combined with the MobileNet architecture are designed specifically for mobile and embedded vision. They trade a small amount of accuracy for a massive boost in speed.

3. OpenCV Implementation

The secret to speed is avoiding heavy frameworks like PyTorch or TensorFlow at runtime. Instead, we use OpenCV's built-in DNN module which is highly optimized for C++ and CPU instructions.

# Load the optimized Caffe model

4. Performance Tuning

To hit the stable 30fps target, I had to apply a few additional optimizations:

  • Multi-threading: Reading frames from the camera in a separate thread (using `imutils.VideoStream`) prevents I/O blocking.
  • Frame Skipping: If the detection queue is full, skip frames rather than lagging behind real-time.
  • Resolution: Processing at 320x240 and then scaling coordinates back up for display significantly reduces CPU load.

5. Conclusion

While Transformers and Large Vision Models are exciting, specialized efficient architectures like MobileNet still rule the Edge. For under \$50, you can build a smart security system that rivals commercial products.
