VIDES: Vehicle Image Detection System

A production-grade vehicle detection and tracking system designed for building high-quality AI training datasets. VIDES handles multiple RTSP camera streams simultaneously with advanced tracking algorithms to prevent duplicates and sophisticated quality control mechanisms.

Key Features:
  • Multi-camera RTSP stream support with independent processing
  • ByteTrack + Kalman filtering for zero-duplicate tracking
  • ROI-based filtering with custom polygon drawing
  • Quality control: blur detection, brightness check, stationary vehicle filtering
  • Real-time speed estimation and direction detection
  • Custom YOLO model supporting 7 vehicle classes (auto_rickshaw, bike, bus, car, mini_bus, tractor, truck)
  • High-performance multi-threaded architecture
  • Organized dataset generation with camera-wise folder structure
Technologies:

  • Python with multi-threading for optimal performance
  • Kalman filters for trajectory smoothing
  • NumPy for numerical computations
Applications:

DocuAI Studio v3.1 - Document AI + RAG Pipeline

A production-ready, locally hosted Document AI system that supports multi-format ingestion and grounded AI answers with citations. The platform combines semantic retrieval and keyword retrieval with reranking for high-quality responses.

README Highlights:
  • Hybrid retrieval pipeline: FAISS + BM25 + Reciprocal Rank Fusion (RRF)
  • 40+ REST/WebSocket endpoints on FastAPI
  • Supports PDF, DOCX, XLSX, PPTX, CSV, image OCR, text, and web URL ingestion
  • Streaming chat, batch Q&A, semantic search, and source-grounded answers
  • Knowledge graph explorer, query analytics dashboard, and document versioning
  • Index export/import, scheduled crawling, and async ingestion with task tracking
  • Runs fully local with Ollama (Mistral) and supports OpenAI backend via env config
Architecture Snapshot:
  • Document -> OCR/Parse -> Semantic Chunking -> Embeddings (MiniLM) -> FAISS index
  • Query -> FAISS search -> BM25 merge + RRF -> Cross-encoder rerank -> LLM response
  • Session persistence, cache layer, analytics tracking, and WebSocket collaboration
Tech Stack:
  • Backend: Python, FastAPI, LangChain, FAISS, SQLite, Docker
  • LLM: Ollama (default) with optional OpenAI provider support
  • Frontend: HTML, CSS, JavaScript with markdown rendering and mobile-optimized UI
  • Testing/Quality: Pytest suite, CI workflow, Makefile, pinned requirements
Why This Project Matters:
  • Demonstrates end-to-end AI product engineering, not just notebook experiments
  • Balances local privacy-first deployment with cloud-LLM flexibility
  • Includes production concerns: security hardening, async pipelines, observability, and backup/restore
